## 目标

将仓库根目录的 `README.md` 重写为同时服务剧场运营人员和开源开发者的项目首页。读者应能在开篇快速理解项目解决的问题和已实现功能，并能在后续章节中完成本地启动、测试和二次开发。

## 目标读者

- 剧场运营和排班人员：关心业务价值、功能范围和操作流程。

## 文档结构

README 采用“产品价值优先、开发信息分层”的顺序：

2. 适用场景和项目价值。
3. 已实现功能，按管理员端、演员端、导入流程和排班引擎分组。
4. V1 业务流程：基础配置、月度计划、指定/许愿导入、周排班预览。
5. 技术栈和仓库目录。
6. MySQL、FastAPI 后端和 React 前端的快速启动步骤。
7. 演示账号、API 文档和测试命令。
8. 开发状态、已知边界、贡献方式、MIT 许可证和设计文档链接。

## 内容准则

- 以当前代码、测试和验收记录为准，不把设计文档中的未完成能力写成已交付功能。
- 明确说明当前是 V1，周排班页的完整操作台、发布和导出仍属于后续工作。
- 启动命令仅使用仓库已支持的 Python、Alembic、MySQL、npm 和 Vite 流程。
- 不添加无法验证的在线演示、截图、CI 状态或覆盖率；项目所有者已选择 MIT License。
- 保留中文为主的表达，技术名称和命令保持原文。

## 开篇功能总结

开篇用一段简洁文字说明：这是一个面向剧场卡司管理的开源排班系统，贯通剧场、角色、演员能力和请假管理，支持按月生成场次、按周导入指定与许愿，并在硬性规则、指定优先级和许愿偏好之间生成可解释的排班预览。

随后用紧凑的功能列表强调：

- 剧场、角色、演员与跨卡能力管理。
- 月度场次生成、公休和整天请假审核。
- 指定/许愿文本半自动解析、校验、人工修正和幂等确认。
- 硬规则过滤、指定优先级、许愿偏好和空槽/失败原因输出。
- 管理员与演员角色登录，演员可查看排班并提交请假。

## 快速启动设计

README 将使用本地开发流程：

1. 创建 MySQL 8.0 数据库 `theater_cast_scheduling`。
2. 在 `backend` 中创建 Python 3.11+ 虚拟环境并安装 `.[dev]`。
3. 通过环境变量配置 `DATABASE_URL` 和 `JWT_SECRET`。
4. 运行 `alembic upgrade head` 和 `uvicorn app.main:app --reload`。
5. 在 `frontend` 中运行 `npm install` 和 `npm run dev`。
6. 通过 `pytest -q`、`ruff check .`、`npm run test` 和 `npm run build` 验证开发环境。

为使分别运行在 `8000` 和 `5173` 端口的后端与前端可直接联调，FastAPI 仅允许本地 Vite 开发来源 `http://localhost:5173` 和 `http://127.0.0.1:5173`，不使用通配跨域来源。
